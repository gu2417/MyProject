# 요구사항 분석서

## 1. 문서 개요

본 문서는 `docs/overview/requirements_analysis.md`를 1순위 기준으로 하고, `docs/development/development_plan.md`를 구현 구조 및 개발 방향 참고 기준으로 반영하여 작성한 학생관리 시스템의 단일 최종 요구사항 문서이다.

기존 `requirements.md`의 본문 내용은 재사용하지 않으며, 문서의 제목 구조, 번호 체계, 표 형식, 요구사항 ID 중심의 작성 방식만 유지한다.

본 시스템은 C 언어 기반 Windows 콘솔 프로그램이며, 서버 프로그램과 클라이언트 프로그램을 분리하여 구성한다. 클라이언트는 로그인, 메뉴 출력, 입력 수집, 결과 출력을 담당하고, 서버는 데이터 로딩, 요청 처리, 파일 저장, 연결 리스트 기반 학생 데이터 관리를 담당한다.

## 2. 시스템 목적

본 시스템은 교직원이 학생 정보를 효율적으로 관리할 수 있도록 지원하는 C 언어 기반 Windows 콘솔 학생관리 프로그램이다.

사용자는 클라이언트 프로그램에서 로그인한 뒤 서버에 학생 정보 및 사용자 관리 요청을 전송한다. 서버는 파일에서 데이터를 읽고, 학생 데이터를 연결 리스트로 관리하며, 변경 사항을 파일에 저장한다.

## 3. 사용자 및 이해관계자

| 구분 | 설명 |
| --- | --- |
| 일반 사용자 | 로그인 후 학생 정보를 조회, 추가, 수정, 삭제하는 교직원 |
| 사용자 관리자 | 사용자 계정을 추가하거나 삭제하는 교직원 |
| 서버 프로그램 | 데이터 로딩, 요청 처리, 파일 저장 담당 |
| 클라이언트 프로그램 | 로그인, 메뉴 출력, 입력 수집, 결과 출력 담당 |

## 4. 시스템 범위

### 4.1 포함 범위

- Windows PC에서 동작하는 C 언어 콘솔 프로그램
- 서버 프로그램과 클라이언트 프로그램 분리
- 소켓 통신 기반 서버-클라이언트 요청 및 응답 처리
- 사용자 로그인 인증
- 사용자 계정 추가 및 삭제
- 학생 정보 조회, 추가, 수정, 삭제
- 학과명 기준 학생 목록 조회
- 단과대학과 소속 학과 정보를 이용한 학생 검색
- `users.txt` 기반 사용자 정보 관리
- `SMU_Students.txt` 기반 학생 정보 관리
- 연결 리스트 기반 학생 데이터 관리
- 파일 입출력 기반 데이터 로딩 및 저장
- `time.h` 라이브러리 사용
- 서버/클라이언트/공통 모듈 파일 분리
- 헤더 파일 기반 구조체, 상수, 함수 원형 분리

### 4.2 제외 범위

- GUI 프로그램
- 데이터베이스 서버 사용
- 웹 또는 모바일 애플리케이션
- 외부 인증 시스템 연동
- 수강신청, 성적 산출, 등록금 관리 등 학사 행정 전체 기능
- 요구사항에 없는 AI 기능

## 14. 필수 구조체 후보

아래 구조체는 요구사항을 만족하기 위한 분석 단계의 후보이며, 실제 구현 시 C 문법과 파일 분리 구조에 맞게 구체화한다. 필드 크기, 파일명, 헤더 파일명, 함수 원형은 요구사항 미정이다.

```c
typedef struct Student {
    char student_id[20];
    char name[50];
    char department[80];
    int grade;
    double gpa;
    char email[100];
    char phone[30];
    struct Student *next;
} Student;

typedef struct User {
    char user_id[40];
    char password[80];
    char department[80];
    char name[50];
} User;
```

단과대학과 소속 학과 정보를 표현하기 위한 구조체는 필요하나, 구체적인 구조체 이름과 필드 구성은 요구사항 미정이다.
