# 오류 및 복구 화면

네트워크 단절·프로토콜 오류·세션 만료 등 예외 상황의 화면 처리와 복구 흐름을 정의한다.

---

## 1. 서버 연결 끊김 화면

`recv()` 반환값이 0 또는 음수인 경우 RecvMsg 스레드에서 감지:

```
============================================================
    ⚠  연결 끊김 (Connection Lost)
============================================================

  서버와의 연결이 끊어졌습니다.

  가능한 원인:
    • 서버가 종료되었습니다
    • 네트워크가 불안정합니다
    • 세션이 비정상적으로 종료되었습니다

------------------------------------------------------------
  1. 재연결 시도
  2. 프로그램 종료
  0. 종료 (기본)

> Enter Command:
```

- **1**: `connect()` 재시도 → 성공 시 자동 로그인 시도
- **2 / 0**: `closesocket()`, `WSACleanup()` 후 종료

```c
void on_connection_lost(SOCKET sock) {
    closesocket(sock);
    show_disconnect_screen();
    int cmd = get_user_command();
    if (cmd == 1) attempt_reconnect();
    else exitService(0);
}
```

## 2. 재연결 시도 화면

```
============================================================
    [재연결 시도 중...]
============================================================

  서버 주소: 127.0.0.1:55555
  시도 횟수: 1 / 3

  연결 중...

  [성공] 서버에 다시 연결되었습니다.
  > 자동 로그인 시도 중... (id: alice)
  [성공] 로그인 완료. 마지막 화면으로 복귀합니다.
```

실패 케이스:
```
  [실패] 서버에 연결할 수 없습니다. (시도 1/3)
  [실패] 서버에 연결할 수 없습니다. (시도 2/3)
  [실패] 서버에 연결할 수 없습니다. (시도 3/3)

  ※ 잠시 후 다시 시도해주세요.
> Enter 키를 눌러 종료:
```

```c
int attempt_reconnect(void) {
    for (int i = 1; i <= 3; i++) {
        printf("  연결 중... (시도 %d/3)\n", i);
        SOCKET s = connect_to_server();
        if (s != INVALID_SOCKET) {
            g_state.sock = s;
            auto_login();
            return 1;
        }
        Sleep(2000);
    }
    return 0;
}
```

## 4. 세션 만료 화면 (P1 이후)

장시간 미응답 또는 PING 실패 시:

```
============================================================
    ⚠  세션 만료
============================================================

  서버로부터 응답이 없어 세션이 만료되었습니다.
  마지막 응답: 60초 전

------------------------------------------------------------
  1. 다시 로그인
  0. 종료

> Enter Command:
```

PING/PONG 구현 (P2):
```c
/* 30초마다 PING 전송, 60초 응답 없으면 만료 */
void *ping_thread(void *arg) {
    while (g_state.connected) {
        Sleep(30000);
        send_packet(g_state.sock, "PING|");
        time_t sent = time(NULL);
        while (time(NULL) - sent < 60) {
            if (g_state.last_pong > sent) break;
            Sleep(1000);
        }
        if (g_state.last_pong <= sent) {
            on_session_expired();
            break;
        }
    }
    return NULL;
}
```

## 6. 권한 부족 오류

방장/관리자만 가능한 명령을 일반 멤버가 시도:

```
> 메시지 입력: /kick charlie
  [오류] 강퇴 권한이 없습니다. (방장 또는 관리자만 가능)

```

```
> 메시지 입력: /edit 42 변경 시도
  [오류] 본인이 작성한 메시지만 수정할 수 있습니다.
```

```
  [오류] 수정 가능 시간(5분)이 초과되었습니다.
```

## 7. 서버 응답 타임아웃

특정 요청에 대한 응답이 일정 시간 없을 때:

```
> 친구 추가 요청 전송 중... (응답 대기)

  [경고] 응답이 5초 이상 지연되고 있습니다.
  서버가 처리 중일 수 있습니다.

  1. 계속 대기
  2. 취소

> Enter Command:
```

```c
int wait_response_with_timeout(int timeout_sec) {
    time_t start = time(NULL);
    while (!g_state.response_received) {
        if (time(NULL) - start > timeout_sec) {
            return 0;  /* 타임아웃 */
        }
        Sleep(100);
    }
    return 1;
}
```

## 12. 오류 코드 매핑 표

| 코드 | 화면 | 복구 동작 |
|------|------|-----------|
| `E_CONN_LOST` | 1. 연결 끊김 | 재연결 시도 |
| `E_RECONNECT_FAIL` | 2. 재연결 실패 | 종료 |
| `E_PACKET_OVERFLOW` | 3. 프로토콜 오류 | 강제 종료 |
| `E_PACKET_UNKNOWN` | 3. 경고 인라인 | 무시 후 계속 |
| `E_SESSION_EXPIRED` | 4. 세션 만료 | 재로그인 또는 종료 |
| `E_FORBIDDEN_CHAR` | 5-1. 금지 문자 | 재입력 |
| `E_LENGTH_EXCEED` | 5-2. 길이 초과 | 재입력 |
| `E_REQUIRED_EMPTY` | 5-3. 빈 입력 | 재입력 |
| `E_PW_MISMATCH` | 5-4. 비번 불일치 | 처음부터 |
| `E_PERM_DENIED` | 6. 권한 부족 | 명령 무시 |
| `E_EDIT_TIMEOUT` | 6. 수정 시간 초과 | 명령 무시 |
| `E_RESP_TIMEOUT` | 7. 응답 타임아웃 | 대기/취소 선택 |
| `E_SEND_FAIL` | 8. 전송 실패 | 재연결 시도 |
| `E_MALLOC_FAIL` | 9. 메모리 부족 | 강제 종료 |
| `E_FORCE_LOGOUT` | 10. 강제 로그아웃 | 종료 |

## 13. 통합 오류 출력 매크로

```c
#define ERR(code, msg)  printf("\n  [오류] %s (%s)\n> ", msg, code); fflush(stdout)
#define WARN(msg)       printf("\n  [경고] %s\n> ", msg); fflush(stdout)
#define INFO(msg)       printf("\n  [알림] %s\n> ", msg); fflush(stdout)

/* 사용 예 */
if (input_len > MAX_CONTENT_LEN) {
    ERR("E_LENGTH_EXCEED", "메시지 길이 초과");
    return -1;
}

if (!g_state.connected) {
    ERR("E_CONN_LOST", "서버 연결이 끊어졌습니다");
    on_connection_lost(g_state.sock);
}
```
