# 다이얼로그 및 액션 메뉴

콘솔 환경에서의 확인 다이얼로그와 액션 컨텍스트 메뉴 패턴을 정리한다. 모든 다이얼로그는 입력 줄에 인라인으로 표시되며, `0` 또는 `n` 입력으로 취소한다.

---

## 1. 로그아웃 확인 다이얼로그

```
============================================================
    [로그아웃 확인]
============================================================

  현재 접속 세션을 종료합니다.
  진행 중인 채팅이 있다면 모든 화면이 닫힙니다.

> 로그아웃하시겠습니까? (y/n):
```

- `y` → `LOGOUT_REQ` 전송 (P1) 또는 소켓 종료 (P0) → 초기 화면 복귀
- `n` 또는 `0` → 메인 로비 복귀

---

## 2. 채팅방 나가기 확인 다이얼로그 (FR-G05)

```
============================================================
    [채팅방 나가기 확인]
============================================================

  방: 개발팀
  현재 방에서 퇴장합니다.
  ※ 마지막 멤버일 경우 방이 자동 삭제됩니다.

> 정말 나가시겠습니까? (y/n):
```

- `y` → `ROOM_LEAVE|<room_id>` 전송 → 메인 로비 복귀
- `n` → 채팅방 내부로 복귀

---

## 3. 멤버 강퇴 확인 다이얼로그 (FR-G06)

```
============================================================
    [멤버 강퇴 확인]
============================================================

  방: 개발팀
  대상: charlie (박민준)
  ※ 강퇴된 멤버는 즉시 방에서 제거되며,
    같은 방에 다시 입장할 수 있습니다.

> 강퇴를 진행하시겠습니까? (y/n):
```

- `y` → `ROOM_KICK|<room_id>:<target_id>` 전송
  - 시스템 메시지 브로드캐스트: `--- 박민준 님이 강퇴되었습니다. ---`
  - 강퇴 대상에게 `ROOM_KICKED_NOTIFY` 전송 → 강제 퇴장
- `n` → 멤버 액션 메뉴로 복귀

---

## 4. 방 삭제 확인 다이얼로그 (FR-G06)

```
============================================================
    [방 삭제 확인]  ⚠ 복구 불가
============================================================

  방: 개발팀
  현재 멤버: 3명
  메시지 수: 1,234건

  ※ 모든 메시지·멤버 정보가 영구 삭제됩니다.
  ※ 모든 멤버가 메인 로비로 강제 이동됩니다.

> 정말 삭제하시겠습니까? (yes 입력):
```

- 단순 `y`가 아닌 `yes`를 입력해야 진행 (실수 방지)
- `yes` → `ROOM_DELETE|<room_id>` 전송 → `ROOM_DELETED_NOTIFY` 전체 멤버에게 전송
- 그 외 → 채팅방으로 복귀

  ※ 진행 중인 DM 대화는 유지됩니다.

> 친구를 삭제하시겠습니까? (y/n):
```

## 6. 친구 차단 확인 다이얼로그 (FR-F05)

============================================================

  ※ 차단 시 다음이 적용됩니다:

    - 상대방의 친구 추가 시도 차단

```

## 7. 메시지 삭제 확인 다이얼로그 (FR-M02)

```
============================================================
    [메시지 삭제 확인]
============================================================

  메시지 #42
  내용: "오늘 회의 자료 준비됐나요?"

  ※ 다른 멤버에게는 [삭제된 메시지]로 표시됩니다.
  ※ 삭제 후 복구할 수 없습니다.

> 삭제하시겠습니까? (y/n):
```

- `y` → `MSG_DELETE|<room_id>:<msg_id>` 전송
- 채팅방 전체에 `MSG_DELETED_NOTIFY` 브로드캐스트

## 8. 메시지 수정 모드 화면 (FR-M03)

```
============================================================
    [메시지 수정]  메시지 #42  (3분 25초 경과)
============================================================

  현재 내용:
    "오늘 회의 자료 준비됐나요?"

> 새 내용 입력 (Enter=취소, max 500자):
  오늘 회의 자료 준비되셨나요?

  [성공] 메시지가 수정되었습니다. (수정됨) 표시가 추가됩니다.
```

5분 초과 케이스:
```
  [오류] 수정 가능 시간(5분)이 초과되었습니다.
  > 채팅방으로 복귀합니다...
```

- 명령어: `/edit <msg_id> <새내용>` 또는 메시지 액션 메뉴 → "수정"
- 서버: 5분(300초) 검증 → `MSG_EDIT|<room_id>:<msg_id>:<new_content>`
- 응답 후 채팅방 전체에 `MSG_EDITED_NOTIFY` 브로드캐스트

## 9. 답장(인용) 작성 화면 (FR-M04)

```
============================================================
    [답장 작성]  → 메시지 #42
============================================================

  ┌── 인용 원문 ──────────────────────────────────────────┐
  │  김철수 [14:30]                                       │
  │  오늘 회의 자료 준비됐나요?                            │
  └────────────────────────────────────────────────────────┘

> 답장 내용 입력 (Enter=취소, max 500자):
  네, 이미 공유 폴더에 올려두었습니다!

  [성공] 답장이 전송되었습니다.
```

- 명령어: `/reply <msg_id> <내용>` 또는 메시지 액션 메뉴 → "답장"
- 서버: `MSG_REPLY|<room_id>:<reply_to_id>:<content>` → `ROOM_MSG_RECV` 브로드캐스트 (reply_to_id 포함)

## 10. 메시지 액션 컨텍스트 메뉴

채팅방 내부에서 `/select <msg_id>` 또는 메시지 호버 동작 입력 시:

```
============================================================
    [메시지 #42 액션]
============================================================

  김철수 [14:30]
  "오늘 회의 자료 준비됐나요?"

------------------------------------------------------------
  1. 답장 (/reply)
  2. 수정 (/edit)        ← 본인 메시지 + 5분 이내만
  3. 삭제 (/del)         ← 본인 메시지 또는 방장
  4. 핀 고정 (/pin)      ← 방장/관리자 전용
  5. 귓속말 (/w 닉)      ← 발신자에게 귓속말
  0. 취소

> Enter Command:
```

## 11. 채팅방 친구 초대 화면 (FR-G02)

채팅방 내부에서 `/invite` 입력 시 친구 목록에서 선택:

```
============================================================
    [친구 초대]  방: 개발팀  (3/20명)
============================================================

  초대 가능한 친구 (이미 방 멤버는 제외)
  ────────────────────────────────────────
  1. [ON ] 김철수    (kimcs)
  2. [OFF] 이영희    (leeya)
  3. [바쁨] 박민준   (parkmj)

------------------------------------------------------------
  번호 선택 (여러 명 시 쉼표 구분, 예: 1,3)
  d. ID 직접 입력
  0. 취소

> Enter Command:  1,3

  [성공] 김철수, 박민준 님에게 초대를 전송했습니다.
```

- 단일/다중 선택 모두 지원
- ID 직접 입력 시: `> 초대할 ID 입력: bob`
- 응답 케이스:
  ```
  [알림] 김철수: 이미 방 멤버입니다.        ← ROOM_INVITE_RES|2
  [오류] 박민준: 존재하지 않는 ID입니다.    ← ROOM_INVITE_RES|1
  [오류] 방 인원이 가득 찼습니다.            ← ROOM_INVITE_RES|3
  ```

## 12. 채팅방 초대 수신 알림 + 액션 (FR-G02, P1)

`ROOM_INVITE_NOTIFY|<room_id>:<room_name>:<inviter_nick>` 수신 시:

```
============================================================
    [채팅방 초대]  ↓ 새 알림
============================================================

  alice (홍길동) 님이 방 "개발팀"에 초대했습니다.

  방 정보:
    이름   : 개발팀
    인원   : 3/20명
    주제   : C 언어 프로젝트 팀
    비밀번호: 있음

------------------------------------------------------------
  1. 입장 (수락)
  2. 거절
  0. 나중에 결정 (메인 로비로)

> Enter Command:
```

- 1: 비번 방이면 비밀번호 입력 → `ROOM_JOIN`
- 2: `room_invites.txt` status=2(거절) (구현 시)
- 0: 알림만 닫고 보류 (다음 로그인에 다시 표시)

    [차단 목록]  총 2명
============================================================

  1. eve   (이브)    | 차단일: 2026-04-20
  2. mallet (말렛)   | 차단일: 2026-05-01

------------------------------------------------------------
  번호 선택 → 차단 해제
  0. 뒤로 가기

> Enter Number:  1

  [선택] eve (이브)
  차단을 해제하시겠습니까? (y/n): y

  [성공] eve 님의 차단이 해제되었습니다.

## 17. 핀 메시지 목록 화면 (FR-M10)

채팅방 상단 핀 영역 클릭 또는 `/pinlist` 입력:

    [핀 메시지 목록]  방: 개발팀  (총 3건)
============================================================

  1. [05-01 09:00] alice  : 매주 월요일 9시 회의입니다.

  3. [05-07 11:00] alice  : 다음 회의 수요일 14시로 변경

------------------------------------------------------------
  번호 선택 → 해당 메시지로 이동 / 핀 해제 (방장)
  0. 닫기

> Enter Number:  3

  [선택] 메시지 #58
  1. 해당 메시지로 이동
  2. 핀 해제 (방장 전용)
  0. 취소

> Enter Command:
```

핀 해제 처리:
```
  [성공] 핀이 해제되었습니다.
  ─→ 채팅방 전체에 MSG_PIN_NOTIFY (해제) 브로드캐스트

## 27. 다이얼로그 공통 입력 처리

```c
/* 단순 y/n 확인 */
int confirm(const char *prompt) {
    char buf[16];
    printf("%s (y/n): ", prompt);
    fgets(buf, sizeof(buf), stdin);
    return (buf[0] == 'y' || buf[0] == 'Y');
}

/* 'yes' 전체 입력 확인 (위험 작업) */
int confirm_strict(const char *prompt) {
    char buf[16];
    printf("%s (yes 입력): ", prompt);
    fgets(buf, sizeof(buf), stdin);
    buf[strcspn(buf, "\n")] = '\0';
    return (strcmp(buf, "yes") == 0);
}

/* 다중 선택 입력 (1,3,5 형식) */
int parse_multi_select(const char *input, int *out, int max) {
    int count = 0;
    char tmp[256];
    strncpy(tmp, input, 255); tmp[255] = '\0';
    char *tok = strtok(tmp, ",");
    while (tok && count < max) {
        int n = atoi(tok);
        if (n > 0) out[count++] = n;
        tok = strtok(NULL, ",");
    }
    return count;
}
```
