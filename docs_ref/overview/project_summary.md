# 프로젝트 개요

## 1. 프로젝트 정보

| 항목 | 내용 |
|------|------|
| 프로젝트명 | C Console Chat Application |
| 버전 | 2.0.0 (콘솔 전환) |
| 언어 | C (C11) |
| 아키텍처 | TCP Server-Client, 멀티스레드 |
| 플랫폼 | Windows (WinSock2) |
| 포트 | 55555 |

## 2. 프로젝트 범위

카카오톡의 1:1 채팅/오픈채팅방 구조와 Google Chat의 채널 개념을 **콘솔 터미널** 환경에서 구현한다.  
서버는 최대 256명의 동시 접속을 처리하며, 모든 데이터는 텍스트 파일에 영속 저장한다.

## 3. 기술 결정 근거

### GTK4 제거 → 콘솔 기반 터미널 UI

**이유**:
- `reference/client_main.c` 분석 결과, 기존 구현이 `InitialMenu()` / `ShowMainMenu()` 함수 기반의 콘솔 메뉴 방식으로 작성되어 있음
- `getch()`를 이용한 비밀번호 마스킹 처리가 콘솔 방식으로 구현되어 있음
- GTK4는 Linux/MSYS2 환경 설정이 복잡하며, 순수 C 학습 목적에 불필요한 의존성을 추가함
- 콘솔 방식은 WinSock2만으로 동작하여 빌드 환경 단순화

**콘솔 UI 방식**:
- 번호 메뉴 선택 (`scanf("%d", &cmd)`)
- 비밀번호 입력: `getch()` + `*` 에코 마스킹
- 채팅창: 메시지 수신은 RecvMsg 스레드에서 비동기 출력
- 채팅 명령어: `/leave`, `/members`, `/invite` 등 슬래시 명령어

### MySQL 제거 → 텍스트 파일 기반 영속 저장

**이유**:
- `reference/server_main.c` 분석 결과, `users.txt` 파일을 `id//pw\n` 포맷으로 읽어 인메모리 구조체에 로드하는 방식으로 구현되어 있음
- MySQL 설치/설정 없이 즉시 실행 가능
- 단순한 데이터 구조(사용자, 방, 메시지)에 RDBMS 불필요
- 파일 I/O는 `fopen`/`fgets`/`fprintf`만으로 구현 가능

**파일 저장 방식**:
- 구분자 `//` (reference 파일 준수)
- 서버 시작 시 전체 파일을 인메모리 배열에 로드
- 변경 사항은 즉시 파일에 반영 (append 또는 전체 재작성)
- 파일 없을 때: 빈 배열로 시작 (`fopen()` NULL 체크 필수)

## 4. 채팅방 유형

| 유형 | 구분 | 참여 방식 | is_open 값 |
|------|------|-----------|------------|
| 그룹 채팅방 | 초대 기반 닫힌 방 (카카오톡 단톡방) | 방장/멤버가 친구 초대 | 0 |
| 오픈채팅방 | 누구나 목록에서 발견·참여 가능 | 목록 조회 후 자유 참여 | 1 |
| 1:1 DM | 친구 또는 ID 직접 입력으로 개설 | 자동 개설 | N/A (messages.txt, room_id=0) |

## 5. 기능 범위 (In-Scope / Out-of-Scope)

### In-Scope

| 우선순위 | 기능 |
|----------|------|
| P0 (MVP) | 회원가입, 로그인, 소켓 종료 로그아웃, 방 생성/입장/퇴장, 브로드캐스트 |
| P1 (소셜) | 친구 관리(추가/수락/거절/차단/검색), DM, 초대, 메시지 히스토리, 마이페이지, 정식 로그아웃 |
| P2 (완성도) | 귓속말, 메시지 삭제/수정, 방장 권한, 공지, 알림, 커스터마이징, PING/PONG |
| P3 (풍부한 경험) | 답장, 이모티콘 변환, 검색, 핀 메시지, /me 액션, 타이핑 표시, DND, 멘션 |

### Out-of-Scope

| 기능 | 이유 |
|------|------|
| FR-M05 리액션 | v2.1 이후 별도 구현 |
| FR-ADM01~ADM05 관리자 기능 | 현재 범위 제외 |
| GTK4 GUI | 콘솔로 대체 결정 |
| MySQL | 텍스트 파일로 대체 결정 |
