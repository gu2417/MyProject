# 보안 개요

## 보안 요구사항 (NFR-04)

비밀번호 SHA-256 해시 저장. 평문 전송은 로컬 환경 한정.  
`gets()` 사용 금지 — `fgets()` / `scanf("%Ns", ...)` 필수.

---

## 보안 항목 목록

| 항목 | 위협 | 대책 |
|------|------|------|
| 비밀번호 저장 | 평문 노출 | SHA-256 해시 후 저장 (WinCrypt) |
| 비밀번호 입력 | 화면 노출 | `getch()` + `*` 에코 마스킹 |
| 버퍼 오버플로우 | 스택 스매싱 | `fgets()`, 고정 버퍼 크기 상수 사용 |
| 프로토콜 인젝션 | 패킷 파싱 오류 | 금지문자 사전 검증 (`:`, `;`, `\|`) |
| 중복 로그인 | 세션 탈취 | active 세션 체크 후 ALREADY_ONLINE |
| 경쟁 조건 | 메모리 오염 | CreateMutex / WaitForSingleObject |
| 비정상 종료 | 서버 크래시 | recv() 반환값 체크, leftClient() 처리 |

---

## 위협 모델

```
[공격자] → [네트워크] → [서버]
    ↓
1. 패킷 조작        → 금지문자 검증으로 차단
2. 버퍼 오버플로우  → fgets + 고정 크기 버퍼
3. 비밀번호 도용    → SHA-256 해시 저장
4. 세션 탈취        → 중복 로그인 차단
5. DoS              → MAX_CLIENTS=256 제한
```

> **범위 제한**: 로컬 네트워크 환경 기준. TLS/암호화 채널은 Out-of-Scope.
