# 기능 명세

## 1. 로그인

| ID | 기능 | 설명 |
|------|------|------|
| FR-AUTH-001 | 로그인 입력 | 학생 관리자는 클라이언트에서 사용자ID와 사용자암호를 입력한다. |
| FR-AUTH-002 | 로그인 요청 | 클라이언트는 로그인 요청을 서버에 송신한다. |
| FR-AUTH-003 | 로그인 검증 | 서버는 `users.txt` 기준으로 사용자ID와 사용자암호를 검증한다. |
| FR-AUTH-004 | 권한 부여 | 로그인된 인가 사용자만 학생정보 관리 기능을 사용할 수 있다. |
| FR-AUTH-005 | 결과 출력 | 클라이언트는 서버 로그인 결과를 출력한다. |

## 2. 신규 사용자 추가

| ID | 기능 | 설명 |
|------|------|------|
| FR-USER-ADD-001 | 권한 확인 | 로그인된 사용자만 신규 사용자를 추가할 수 있다. |
| FR-USER-ADD-002 | 사용자 정보 입력 | 사용자ID, 사용자암호, 사용자 소속부서, 사용자 이름을 입력한다. |
| FR-USER-ADD-003 | 파일 반영 | 서버는 신규 사용자 정보를 `users.txt`에 탭 구분 형식으로 저장한다. |
| FR-USER-ADD-004 | 중복 확인 | 사용자ID는 중복될 수 없다. |

## 3. 사용자 삭제

| ID | 기능 | 설명 |
|------|------|------|
| FR-USER-DEL-001 | 삭제 요청 | 사용자 삭제 기능을 제공한다. |
| FR-USER-DEL-002 | 퇴사자 고려 | 퇴사자 계정 삭제를 고려한다. |
| FR-USER-DEL-003 | 파일 반영 | 삭제 결과는 `users.txt`에 반영한다. |
| FR-USER-DEL-004 | 식별 기준 | 삭제 대상 식별 기준은 요구사항 미정이다. |

## 4. 학생 검색

| ID | 기능 | 설명 |
|------|------|------|
| FR-STU-SEARCH-001 | 검색 요청 | 로그인된 사용자는 학생 검색 기능을 사용할 수 있다. |
| FR-STU-SEARCH-002 | 검색 처리 | 서버는 연결 리스트 기반으로 학생 검색 요청을 처리한다. |
| FR-STU-SEARCH-003 | 결과 출력 | 클라이언트는 서버 검색 결과를 출력한다. |
| FR-STU-SEARCH-004 | 검색 조건 | 학생 검색 조건은 요구사항 미정이다. |

## 5. 학생 추가

| ID | 기능 | 설명 |
|------|------|------|
| FR-STU-ADD-001 | 정보 입력 | 이름, 학번, 소속학과, 학년, GPA, 이메일 연락처, 모바일 폰 연락처를 입력한다. |
| FR-STU-ADD-002 | 리스트 반영 | 서버는 신규 학생 정보를 연결 리스트에 추가한다. |
| FR-STU-ADD-003 | 파일 반영 | 추가된 학생 정보는 `SMU_Students.txt`에 반영한다. |

## 6. 학생 삭제

| ID | 기능 | 설명 |
|------|------|------|
| FR-STU-DEL-001 | 삭제 요청 | 로그인된 사용자는 학생 삭제 기능을 사용할 수 있다. |
| FR-STU-DEL-002 | 리스트 반영 | 서버는 삭제 요청을 연결 리스트에 반영한다. |
| FR-STU-DEL-003 | 파일 반영 | 삭제된 학생 정보는 `SMU_Students.txt`에 반영한다. |
| FR-STU-DEL-004 | 식별 기준 | 삭제 대상 식별 기준은 요구사항 미정이다. |

## 7. 학생 정보 수정

| ID | 기능 | 설명 |
|------|------|------|
| FR-STU-UPD-001 | 수정 요청 | 로그인된 사용자는 학생 정보 수정 기능을 사용할 수 있다. |
| FR-STU-UPD-002 | 리스트 반영 | 서버는 수정 요청을 연결 리스트에 반영한다. |
| FR-STU-UPD-003 | 파일 반영 | 수정된 학생 정보는 `SMU_Students.txt`에 반영한다. |
| FR-STU-UPD-004 | 수정 기준 | 수정 대상 식별 기준과 수정 가능 필드는 요구사항 미정이다. |

## 8. 학과명 기반 전체 학생 조회

| ID | 기능 | 설명 |
|------|------|------|
| FR-DEPT-001 | 학과명 입력 | 로그인된 사용자는 학과명을 입력한다. |
| FR-DEPT-002 | 목록 조회 | 서버는 소속학과가 일치하는 전체 학생을 연결 리스트에서 찾는다. |
| FR-DEPT-003 | 결과 출력 | 클라이언트는 해당 학과 학생 목록을 출력한다. |

## 9. 매일 01:00 학생정보 저장

| ID | 기능 | 설명 |
|------|------|------|
| FR-SAVE-001 | 저장 시점 | 서버는 매일 01:00에 학생정보 저장을 수행한다. |
| FR-SAVE-002 | 시간 처리 | 저장 시점 판단에는 `time.h`를 사용한다. |
| FR-SAVE-003 | 파일 반영 | 추가, 수정, 삭제된 학생 데이터를 `SMU_Students.txt`에 반영한다. |

## 10. 서버-클라이언트 메시지

| ID | 기능 | 설명 |
|------|------|------|
| NET-001 | 요청 송신 | 클라이언트는 사용자 명령 기반 요청을 서버에 송신한다. |
| NET-002 | 결과 반환 | 서버는 처리 결과를 클라이언트에 반환한다. |
| NET-003 | 메시지 포맷 | 메시지는 `TYPE<TAB>COMMAND<TAB>STATUS<TAB>PAYLOAD` 형식을 따른다. |
