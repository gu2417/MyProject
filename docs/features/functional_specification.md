# 기능 명세

## 1. 로그인

| ID | 기능 | 설명 |
|------|------|------|
| FR-AUTH-001 | 로그인 입력 | 사용자는 클라이언트에서 사용자 ID와 비밀번호를 입력한다. |
| FR-AUTH-002 | 인증 요청 | 클라이언트는 로그인 요청을 서버에 전송한다. |
| FR-AUTH-003 | 계정 검증 | 서버는 `users.txt` 기반 사용자 정보로 인증한다. |
| FR-AUTH-004 | 결과 반환 | 서버는 로그인 성공 또는 실패 결과를 클라이언트에 전달한다. |
| FR-AUTH-005 | 권한 제한 | 로그인하지 않은 사용자는 학생 정보 관리와 사용자 계정 관리를 사용할 수 없다. |

## 2. 사용자 계정 추가

| ID | 기능 | 설명 |
|------|------|------|
| FR-USER-ADD-001 | 권한 확인 | 로그인한 사용자만 신규 사용자 계정을 추가할 수 있다. |
| FR-USER-ADD-002 | 필드 입력 | 사용자 ID, 비밀번호, 소속 부서, 이름을 입력한다. |
| FR-USER-ADD-003 | 중복 확인 | 서버는 신규 사용자 ID가 기존 ID와 중복되는지 확인한다. |
| FR-USER-ADD-004 | 입력 검증 | 빈 필드는 입력 오류로 처리한다. |
| FR-USER-ADD-005 | 저장 | 추가 성공 시 `users.txt`에 저장한다. |

## 3. 사용자 계정 삭제

| ID | 기능 | 설명 |
|------|------|------|
| FR-USER-DEL-001 | 권한 확인 | 로그인한 사용자만 사용자 계정을 삭제할 수 있다. |
| FR-USER-DEL-002 | 삭제 기준 | 사용자 삭제 기준은 요구사항 미정이다. |
| FR-USER-DEL-003 | 파일 반영 | 삭제 성공 시 `users.txt`에 반영한다. |
| FR-USER-DEL-004 | 삭제 제한 | 현재 로그인 사용자 삭제 제한 여부는 요구사항 미정이다. |

## 4. 학생 정보 조회

| ID | 기능 | 설명 |
|------|------|------|
| FR-STU-SEARCH-001 | 조회 요청 | 로그인한 사용자는 학생 정보를 조회할 수 있다. |
| FR-STU-SEARCH-002 | 조회 조건 | 학생 조회 조건의 세부 항목은 요구사항 미정이다. |
| FR-STU-SEARCH-003 | 결과 출력 | 조회 결과는 클라이언트 화면에 출력한다. |

## 5. 학과명 기준 학생 목록 조회

| ID | 기능 | 설명 |
|------|------|------|
| FR-DEPT-001 | 학과명 입력 | 로그인한 사용자는 학과명을 기준으로 학생 목록을 조회할 수 있다. |
| FR-DEPT-002 | 목록 검색 | 서버는 학생 연결 리스트에서 소속 학과가 일치하는 학생을 찾는다. |
| FR-DEPT-003 | 결과 출력 | 조회 결과는 클라이언트 화면에 출력한다. |

## 6. 단과대학 기준 학생 검색

| ID | 기능 | 설명 |
|------|------|------|
| FR-COLLEGE-001 | 검색 요청 | 로그인한 사용자는 단과대학과 소속 학과 정보를 이용해 학생을 검색할 수 있다. |
| FR-COLLEGE-002 | 검색 조건 | 서버는 단과대학과 소속 학과 정보를 검색 조건으로 사용한다. |
| FR-COLLEGE-003 | 미정 항목 | 단과대학 및 소속 학과 정보의 구체적인 데이터 구조와 초기값은 요구사항 미정이다. |

## 7. 학생 정보 추가

| ID | 기능 | 설명 |
|------|------|------|
| FR-STU-ADD-001 | 필드 입력 | 학번, 이름, 학과, 학년, GPA, 이메일, 전화번호를 입력한다. |
| FR-STU-ADD-002 | 검증 | 서버는 빈 필드, 범위 오류, 학번 중복 여부를 검증한다. |
| FR-STU-ADD-003 | 리스트 삽입 | 검증 성공 시 새 학생 노드를 연결 리스트에 삽입한다. |
| FR-STU-ADD-004 | 파일 저장 | 변경 사항을 `SMU_Students.txt`에 저장한다. |

## 8. 학생 정보 수정

| ID | 기능 | 설명 |
|------|------|------|
| FR-STU-UPD-001 | 수정 요청 | 로그인한 사용자는 학생 정보를 수정할 수 있다. |
| FR-STU-UPD-002 | 대상 식별 | 학생 수정 대상 식별 기준은 요구사항 미정이다. |
| FR-STU-UPD-003 | 파일 저장 | 수정 성공 시 연결 리스트와 `SMU_Students.txt`에 반영한다. |
| FR-STU-UPD-004 | 일부 수정 | 일부 필드 수정 허용 여부는 요구사항 미정이다. |

## 9. 학생 정보 삭제

| ID | 기능 | 설명 |
|------|------|------|
| FR-STU-DEL-001 | 삭제 요청 | 로그인한 사용자는 학생 정보를 삭제할 수 있다. |
| FR-STU-DEL-002 | 삭제 기준 | 학생 삭제 기준은 요구사항 미정이다. |
| FR-STU-DEL-003 | 파일 저장 | 삭제 성공 시 연결 리스트와 `SMU_Students.txt`에 반영한다. |

## 10. 파일 저장

| ID | 기능 | 설명 |
|------|------|------|
| FR-SAVE-001 | 학생 저장 | 학생 정보 변경 내용을 파일에 저장한다. |
| FR-SAVE-002 | 사용자 저장 | 사용자 추가 및 삭제 결과를 파일에 저장한다. |
| FR-SAVE-003 | 저장 실패 | 저장 실패 시 사용자에게 실패 사유를 알린다. |
| FR-SAVE-004 | 시간 처리 | 시간 정보가 필요한 경우 `time.h`를 사용한다. |

## 11. 서버-클라이언트 통신

| ID | 기능 | 설명 |
|------|------|------|
| NET-001 | 요청 전송 | 클라이언트는 사용자 입력을 서버 요청으로 전송한다. |
| NET-002 | 응답 반환 | 서버는 처리 결과를 클라이언트에 반환한다. |
| NET-003 | 명령 분기 | 로그인, 학생 관리, 사용자 관리 요청은 구분되어야 한다. |
| NET-004 | 메시지 형식 | 요청 및 응답 메시지의 구체적인 문자열 형식은 요구사항 미정이다. |
