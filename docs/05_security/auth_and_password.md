# 인증과 사용자암호

## 1. 로그인 기준

서버는 `users.txt` 의 사용자ID와 사용자암호를 기준으로 로그인 요청을 검증한다.

## 2. 사용자 파일

```text
user_id<TAB>password<TAB>department<TAB>name
```

## 3. 요구사항

| ID | 내용 |
|----|------|
| SEC-001 | 로그인되지 않은 사용자는 학생정보 관리 기능을 사용할 수 없다. |
| SEC-002 | 로그인된 사용자만 신규 사용자 추가 기능을 사용할 수 있다. |
| SEC-003 | 사용자 정보는 `users.txt` 에서 관리한다. |

사용자암호 저장 방식의 세부 암호화/해시 정책은 최신 요구사항에 명시되지 않았다.
