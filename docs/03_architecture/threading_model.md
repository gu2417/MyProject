# 실행 루프와 동시성 기준

## 1. 기준

최신 요구사항은 구체적인 스레드 모델을 확정하지 않는다. 구현은 단일 클라이언트 처리 또는 다중 클라이언트 처리 중 하나를 선택할 수 있으나, 서버/클라이언트 소켓 통신과 파일 저장 일관성을 유지해야 한다.

## 2. 공유 자원

| 공유 자원 | 보호 기준 |
|-----------|-----------|
| 학생 연결 리스트 | 학생 추가/삭제/수정/저장 시 동시 접근 방지 |
| `SMU_Students.txt` | 저장 중 다른 쓰기 방지 |
| `users.txt` | 사용자 추가/삭제 중 다른 쓰기 방지 |

## 3. 01:00 저장

매일 01:00 저장은 학생 연결 리스트를 기준으로 수행한다. 저장 중 학생 데이터 변경이 동시에 발생하지 않도록 구현해야 한다.
