# 서버 아키텍처

## 1. 서버 역할

서버는 클라이언트 요청을 수신하고 인증, 학생 관리, 사용자 관리, 파일 저장을 처리한다. 학생 정보는 서버 메모리에서 연결 리스트로 관리한다.

## 2. 서버 파일 구조

```text
src/server/
  server_main.c
  request_handler.c

src/common/
  protocol.h
  student.c
  student.h
  auth.c
  auth.h
  file_io.c
  file_io.h
  validation.c
  validation.h
```

## 3. 모듈별 책임

| 모듈 | 주요 책임 |
|------|------|
| `server_main.c` | Windows 소켓 초기화, bind/listen/accept, 서버 루프 |
| `request_handler.c` | 요청 코드 판별, 기능별 처리 함수 호출 |
| `auth.c/h` | 로그인 인증, 사용자 추가, 사용자 삭제 |
| `student.c/h` | 학생 연결 리스트 관리, 조회, 추가, 수정, 삭제 |
| `file_io.c/h` | `SMU_Students.txt`, `users.txt` 로딩/저장 |
| `validation.c/h` | 빈 값, 범위 오류, 중복 여부, 이메일 형식 검증 |
| `protocol.h` | 요청 코드, 응답 코드, 공통 상수 정의 |

## 4. 서버 시작 흐름

1. 소켓 사용을 위한 Windows 네트워크 초기화를 수행한다.
2. 서버 소켓을 생성하고 포트를 바인딩한다.
3. `SMU_Students.txt`에서 학생 데이터를 읽어 연결 리스트를 구성한다.
4. 필요 시 `users.txt`를 읽어 인증 준비를 수행한다.
5. 클라이언트 연결을 수락한다.
6. 수신 요청을 인증, 학생 관리, 사용자 관리 명령으로 분기한다.

## 5. 요청 처리 흐름

| 요청 분류 | 처리 기준 |
|------|------|
| 로그인 | `users.txt` 기준으로 사용자 ID와 비밀번호 검증 |
| 사용자 추가 | 중복 ID와 빈 필드 검증 후 `users.txt` 저장 |
| 사용자 삭제 | 삭제 기준은 요구사항 미정, 삭제 결과는 `users.txt` 저장 |
| 학생 조회 | 연결 리스트에서 조건에 맞는 학생 검색 |
| 학생 추가 | 필드 검증과 학번 중복 확인 후 연결 리스트 및 파일 반영 |
| 학생 수정 | 대상 식별 기준은 요구사항 미정, 수정 결과는 연결 리스트 및 파일 반영 |
| 학생 삭제 | 삭제 기준은 요구사항 미정, 삭제 결과는 연결 리스트 및 파일 반영 |

## 6. 오류 처리

- 파일 파싱 오류가 발생해도 전체 프로그램이 비정상 종료되지 않도록 처리한다.
- 저장 실패 시 실패 사유를 클라이언트에 반환한다.
- 잘못된 입력은 입력 오류로 처리한다.
- 파일 없음 처리 방식은 요구사항 미정이다.
