# 클라이언트 아키텍처

## 1. 클라이언트 역할

클라이언트는 Windows 콘솔에서 사용자 입력을 받고, 서버에 요청을 전송하며, 서버 응답을 사용자가 읽기 쉬운 형식으로 출력한다.

## 2. 클라이언트 파일 구조

```text
src/client/
  client_main.c
  client_ui.c

src/common/
  protocol.h
  validation.h
```

## 3. 모듈별 책임

| 모듈 | 주요 책임 |
|------|------|
| `client_main.c` | 서버 연결, 클라이언트 실행 루프, 종료 처리 |
| `client_ui.c` | 로그인 입력, 메뉴 출력, 기능별 입력 수집, 결과 출력 |
| `protocol.h` | 요청/응답 코드와 메시지 상수 참조 |
| `validation.h` | 클라이언트 측 기본 입력 검증 참조 |

## 4. 메뉴 범위

| 상태 | 제공 기능 |
|------|------|
| 로그인 전 | 로그인, 종료 |
| 로그인 후 | 학생 조회, 학과명 조회, 단과대학 검색, 학생 추가, 학생 수정, 학생 삭제, 사용자 추가, 사용자 삭제, 종료 |

메뉴 번호와 화면 문구의 구체적인 구성은 요구사항 미정이다.

## 5. 통신 흐름

1. 사용자가 메뉴를 선택한다.
2. 클라이언트가 필요한 입력값을 수집한다.
3. 클라이언트가 요청 코드를 포함한 메시지를 서버에 전송한다.
4. 클라이언트가 서버 응답을 수신한다.
5. 성공, 실패, 조회 결과를 콘솔에 출력한다.

## 6. 출력 원칙

- 결과 출력은 교직원이 이해하기 쉬운 문구로 제공한다.
- 실패 응답은 가능한 경우 실패 사유를 함께 출력한다.
- 비밀번호 표시 방식은 요구사항 미정이다.
