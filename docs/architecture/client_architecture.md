# 클라이언트 아키텍처

## 1. 클라이언트 역할

클라이언트는 학생 관리자가 사용하는 콘솔 프로그램이다. 사용자 명령 기반 요청을 서버에 송신하고, 서버 처리 결과를 출력한다.

## 2. 클라이언트 파일 구조

```text
src/client/
  client_main.c
  client_ui.c

src/common/
  protocol.h
```

## 3. 모듈별 책임

| 모듈 | 주요 책임 |
|------|------|
| `client_main.c` | 서버 접속, 클라이언트 실행 루프 |
| `client_ui.c` | 메뉴 출력, 사용자 입력, 서버 결과 출력 |
| `protocol.h` | 메시지 포맷과 명령 코드 참조 |

## 4. 메뉴 범위

| 상태 | 기능 |
|------|------|
| 로그인 전 | 로그인 |
| 로그인 후 | 학생 검색, 학생 추가, 학생 삭제, 학생 정보 수정, 학과명 기반 전체 학생 조회, 신규 사용자 추가, 사용자 삭제 |

## 5. 요청 흐름

1. 학생 관리자가 메뉴에서 명령을 선택한다.
2. 클라이언트가 필요한 입력값을 수집한다.
3. 클라이언트가 메시지 포맷에 맞게 요청을 구성한다.
4. 클라이언트가 요청을 서버에 송신한다.
5. 클라이언트가 서버 응답을 수신한다.
6. 클라이언트가 서버 결과를 콘솔에 출력한다.
