# 프로젝트 개요

## 1. 프로젝트 정보

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Student Management System |
| 기준 문서 | `requirements.md` |
| 언어 | C |
| 플랫폼 | Windows |
| UI | 콘솔 |
| 아키텍처 | Server-Client |
| 통신 | 소켓 통신 |
| 저장 방식 | 텍스트 파일 입출력 |

## 2. 프로젝트 범위

본 프로젝트는 교직원이 학생 정보를 조회, 추가, 수정, 삭제하고 사용자 계정을 추가 또는 삭제할 수 있는 Windows 콘솔 기반 학생관리 시스템이다.

클라이언트는 입력과 출력, 서버 요청 생성을 담당한다. 서버는 인증, 학생 정보 관리, 사용자 정보 관리, 파일 입출력, 연결 리스트 관리를 담당한다.

## 3. 기술 결정

| 결정 | 내용 |
|------|------|
| 콘솔 UI | GUI 없이 Windows 콘솔 메뉴 기반으로 구현 |
| 파일 저장 | 데이터베이스 없이 `SMU_Students.txt`, `users.txt` 사용 |
| 서버/클라이언트 분리 | 클라이언트는 요청 생성, 서버는 요청 처리 담당 |
| 연결 리스트 | 학생 데이터를 서버 메모리에서 연결 리스트로 관리 |
| `time.h` | 시간 정보 기록 또는 표시가 필요한 경우 사용 |

## 4. 데이터 기준

| 항목 | 기준 |
|------|------|
| 학생 초기 데이터 | 10명 |
| 사용자 초기 데이터 | 3명 |
| 학생 데이터 파일 | `SMU_Students.txt` |
| 사용자 데이터 파일 | `users.txt` |

## 5. 범위 제외

- GUI 프로그램
- 데이터베이스 서버
- 웹 또는 모바일 애플리케이션
- 외부 인증 시스템
- 요구사항에 없는 부가 기능
