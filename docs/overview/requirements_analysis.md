# Requirements Analysis

## 1. 목적

학생관리 시스템은 교직원이 학생 정보를 효율적으로 관리하도록 지원하는 C 언어 기반 Windows 콘솔 프로그램이다. 사용자는 클라이언트 프로그램에서 로그인한 뒤 서버에 학생 정보 및 사용자 관리 요청을 전송한다. 서버는 파일에서 데이터를 읽고, 학생 데이터를 연결 리스트로 관리하며, 변경 사항을 파일에 저장한다.

본 문서는 `requirements.md`의 요약 문서이며, 상세 기준은 항상 `requirements.md`를 따른다.

## 2. 이해관계자

| 구분 | 설명 |
|------|------|
| 일반 사용자 | 로그인 후 학생 정보를 조회, 추가, 수정, 삭제하는 교직원 |
| 사용자 관리자 | 사용자 계정을 추가하거나 삭제하는 교직원 |
| 서버 프로그램 | 데이터 로딩, 요청 처리, 파일 저장 담당 |
| 클라이언트 프로그램 | 로그인, 메뉴 출력, 입력 수집, 결과 출력 담당 |

## 3. 포함 범위

- Windows PC에서 동작하는 C 언어 콘솔 프로그램
- 서버 프로그램과 클라이언트 프로그램 분리
- 소켓 통신 기반 요청 및 응답 처리
- 사용자 로그인 인증
- 사용자 계정 추가 및 삭제
- 학생 정보 조회, 추가, 수정, 삭제
- 학과명 기준 학생 목록 조회
- 단과대학과 소속 학과 정보를 이용한 학생 검색
- `users.txt` 기반 사용자 정보 관리
- `SMU_Students.txt` 기반 학생 정보 관리
- 연결 리스트 기반 학생 데이터 관리
- 파일 입출력 기반 데이터 로딩 및 저장
- `time.h` 라이브러리 사용

## 4. 제외 범위

- GUI 프로그램
- 데이터베이스 서버 사용
- 웹 또는 모바일 애플리케이션
- 외부 인증 시스템 연동
- 수강신청, 성적 산출, 등록금 관리 등 학사 행정 전체 기능
- 요구사항에 없는 AI 기능

## 5. 운영 환경

| ID | 요구사항 |
|------|------|
| ENV-001 | 프로그램은 Windows 운영체제에서 정상 동작해야 한다. |
| ENV-002 | 구현 언어는 C 언어로 한다. |
| ENV-003 | 사용자 인터페이스는 콘솔 기반으로 제공한다. |
| ENV-004 | 서버와 클라이언트는 별도 프로그램으로 구성한다. |
| ENV-005 | 클라이언트는 명령과 데이터를 서버에 전달하고 결과를 출력한다. |
| ENV-006 | 서버는 요청을 구분하여 인증, 학생 관리, 사용자 관리를 처리한다. |
| ENV-007 | 서버와 클라이언트 간 요청 및 응답은 소켓 통신으로 처리한다. |

## 6. 주요 데이터

### 6.1 학생 정보

| 필드 | 설명 | 검증 기준 |
|------|------|------|
| student_id | 학생 고유 학번 | 필수, 중복 불가 |
| name | 학생 이름 | 필수, 빈 값 불가 |
| department | 소속 학과 | 필수, 빈 값 불가 |

| gpa | 평균 학점 | 필수, 일반적으로 0.0~4.5 |
| email | 이메일 주소 | 필수, 기본 이메일 형식 확인 |
| phone | 휴대전화 번호 | 필수, 빈 값 불가 |

|------|------|------|

| password | 로그인 비밀번호 | 필수, 빈 값 불가 |
| department | 사용자 소속 부서 | 필수, 빈 값 불가 |
| name | 사용자 이름 | 필수, 빈 값 불가 |

## 8. 비기능 요구사항

- 서버는 약 9,000명 이상의 학생 데이터를 처리할 수 있어야 한다.
- 파일 파싱 오류가 발생해도 전체 프로그램이 비정상 종료되지 않도록 처리한다.
- 사용자 입력값은 빈 값, 범위 오류, 중복 여부를 검증한다.
- 저장 작업 실패 시 사용자에게 실패 사유를 알린다.
- 메뉴와 결과 출력은 사용자가 이해하기 쉬운 형식으로 제공한다.
- 요구사항에 없는 기능은 추가하지 않는다.
