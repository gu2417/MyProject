# Requirements Analysis

## 1. 목적

대학 교직원을 위한 온라인 학생관리 시스템은 Windows 운영체제를 사용하는 PC에서 동작하는 C 언어 기반 콘솔 프로그램이다. 학생 관리자는 클라이언트 프로그램에서 로그인한 뒤 인가된 사용자로서 학생정보 관리 기능을 사용한다.

본 문서는 `requirements.md`의 요약 문서이며, 상세 기준은 항상 `requirements.md`를 따른다.

## 2. 사용자 및 역할

| 구분 | 설명 |
|------|------|
| 개발요청자 | 대학 교직원을 위한 온라인 학생관리 시스템을 요구한 주체 |
| 학생 관리자 | 로그인 후 학생정보 관리 기능을 사용하는 인가된 사용자 |
| 서버 프로그램 | 학생정보 관리 초기화, 클라이언트 요청 처리, 서비스 제공 |
| 클라이언트 프로그램 | 사용자 명령 기반 요청 송신, 서버 결과 출력 |

## 3. 포함 범위

- C 언어 기반 Windows 콘솔 프로그램
- 학생정보 관리 서버
- 학생 관리자용 클라이언트
- 서버/클라이언트 소켓 통신
- 로그인 기능
- 로그인된 사용자만 학생정보 관리 기능 사용
- 로그인된 사용자만 신규 사용자 추가
- 사용자 삭제
- `users.txt` 기반 사용자 정보 관리
- `SMU_Students.txt` 기반 학생 정보 관리
- 학생 검색, 학생 추가, 학생 삭제, 학생 정보 수정
- 학과명 기반 전체 학생 조회
- 연결 리스트 기반 학생정보 관리
- 매일 01:00 학생정보 저장
- `time.h` 기반 시간 처리
- 송수신용 메시지 포맷 정의
- 서버 기능 구현용 구조체 및 구조체 멤버 정의

## 4. 운영 환경

| ID | 요구사항 |
|------|------|
| ENV-001 | Windows PC에서 정상 동작해야 한다. |
| ENV-002 | C 언어 기반 콘솔 프로그램으로 개발한다. |
| ENV-003 | 서버와 클라이언트는 분리한다. |
| ENV-004 | 서버와 클라이언트는 소켓 통신을 사용한다. |

## 5. 주요 데이터

### 5.1 학생 정보

| 필드 | 설명 |
|------|------|
| name | 이름 |
| student_id | 학번 |
| department | 소속학과 |
| grade | 학년 |
| gpa | GPA(평균학점) |
| email | 이메일 연락처 |
| mobile_phone | 모바일 폰 연락처 |

### 5.2 사용자 정보

| 필드 | 설명 |
|------|------|
| user_id | 사용자ID |
| password | 사용자암호 |
| department | 사용자 소속부서 |
| name | 사용자 이름 |

## 6. 비기능 요구사항

- 서버는 약 9,000명의 학생 수를 기준으로 학생정보를 처리한다.
- 학생정보는 연결 리스트 기반으로 관리한다.
- 학생정보와 사용자 정보는 파일 입출력으로 관리한다.
- 사용자 편의성을 위한 메뉴 및 인터페이스를 제공한다.
- 요구사항에 없는 기능은 추가하지 않는다.
