# 개발 계획

## 1. 개발 목표

`requirements.md`에 맞춰 C 언어 기반 Windows 콘솔 서버-클라이언트 학생관리 시스템을 단계적으로 구현한다. 우선 파일 입출력과 연결 리스트를 안정화한 뒤, 인증과 학생 관리 기능을 결합한다.

## 2. 단계별 계획

| 단계 | 작업 | 산출물 |
|------|------|------|
| 1 | 요구사항 정리 및 데이터 형식 확정 | 요구사항 문서, 파일 스키마 |
| 2 | 기본 프로젝트 구조 작성 | 서버/클라이언트 소스 파일, 헤더 파일 |
| 3 | 학생 연결 리스트 구현 | 학생 추가, 탐색, 수정, 삭제 함수 |
| 4 | 파일 입출력 구현 | `SMU_Students.txt`, `users.txt` 로딩/저장 |
| 5 | 사용자 인증 구현 | 로그인, 사용자 추가, 사용자 삭제 |
| 6 | 클라이언트 메뉴 구현 | 콘솔 입력, 요청 생성, 결과 출력 |
| 7 | 서버 요청 처리 구현 | 명령 분기, 응답 메시지 |
| 8 | 검증 및 예외 처리 | 입력값 검증, 파일 오류 처리 |
| 9 | 통합 테스트 | 정상/오류 시나리오 테스트 |
| 10 | 문서 정리 | 기능 명세, 사용 방법, 테스트 결과 |

## 4. 구현 우선순위

1. 데이터 구조와 파일 입출력
2. 학생 관리 핵심 기능
3. 사용자 인증
4. 클라이언트 메뉴와 서버 요청 처리
5. 예외 처리와 통합 테스트

## 5. 테스트 계획

| 구분 | 테스트 항목 |
|------|------|
| 인증 | 올바른 계정 로그인, 잘못된 비밀번호, 없는 사용자 |
| 사용자 추가 | 정상 추가, 중복 ID, 빈 필드 |
| 사용자 삭제 | 정상 삭제, 없는 ID, 현재 로그인 사용자 삭제 제한 여부 확인 |
| 학생 조회 | 정상 조회, 결과 없음 |
| 학생 추가 | 정상 추가, 중복 학번, GPA 범위 오류, 빈 필드 |
| 학생 수정 | 정상 수정, 대상 없음, 일부 필드 수정 허용 여부 확인 |
| 학생 삭제 | 정상 삭제, 대상 없음 |
| 학과 검색 | 학과별 전체 학생 출력, 결과 없음 |
| 단과대학 검색 | 단과대학 소속 학과 매핑 기반 검색 |
| 파일 처리 | 파일 없음, 필드 부족, 저장 실패 |
| 통신 | 클라이언트 요청 송신, 서버 응답 수신 |

## 6. 완료 기준

- 서버와 클라이언트를 각각 실행할 수 있다.
- 샘플 데이터로 로그인과 학생 조회가 가능하다.
- 학생 추가, 수정, 삭제 결과가 파일에 반영된다.
- 사용자 추가, 삭제 결과가 파일에 반영된다.
- 소켓 통신으로 요청과 응답을 주고받는다.
- 오류 입력에 대해 프로그램이 비정상 종료되지 않는다.
