# 구현 가이드

## 1. 기준

구현은 `requirements.md`를 단일 기준으로 한다. 하위 문서에 더 자세한 설명이 있어도 `requirements.md`와 충돌하면 `requirements.md`를 우선한다.

## 2. 코딩 원칙

- C 언어 기반 Windows 콘솔 프로그램으로 구현한다.
- 서버와 클라이언트를 별도 실행 파일로 분리한다.
- 공통 구조체, 상수, 함수 원형은 헤더 파일에 둔다.
- 학생 데이터는 연결 리스트로 관리한다.
- 데이터 저장은 파일 입출력으로 처리한다.
- 시간 정보가 필요한 경우 `time.h`를 사용한다.

## 3. 파일 분리 원칙

| 파일 | 원칙 |
|------|------|
| `client_main.c` | 클라이언트 실행 진입점 |
| `client_ui.c` | 콘솔 메뉴와 입력 처리 |
| `server_main.c` | 서버 실행 진입점 |
| `request_handler.c` | 서버 요청 분기 |
| `protocol.h` | 요청/응답 코드와 공통 상수 |
| `student.c/h` | 학생 연결 리스트 |
| `auth.c/h` | 인증과 사용자 관리 |
| `file_io.c/h` | 파일 로딩과 저장 |
| `validation.c/h` | 입력 검증 |

## 4. 입력 검증

| 항목 | 기준 |
|------|------|
| 빈 값 | 입력 오류 처리 |
| 학번 | 중복 불가 |
| 사용자 ID | 중복 불가 |
| 학년 | 일반적으로 1~4 |
| GPA | 일반적으로 0.0~4.5 |
| 이메일 | 기본 이메일 형식 확인 |

## 5. 미정 항목 처리

다음 항목은 구현 전에 팀 또는 담당자가 확정해야 한다.

- 요청 및 응답 메시지의 구체적인 문자열 형식
- 파일 레코드 구분자와 필드 순서
- 사용자 삭제 기준
- 학생 수정 및 삭제 대상 식별 기준
- 단과대학 정보의 구체적인 데이터 구조와 초기값
- 비밀번호 저장 방식과 입력 표시 방식
- 파일 없음 처리 방식
