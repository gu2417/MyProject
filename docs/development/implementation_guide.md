# 구현 가이드

## 1. 기준

구현은 `requirements.md`를 단일 최종 기준으로 한다. 최신 요구사항에 없는 기능은 구현하지 않는다.

## 2. 구현 원칙

- C 언어 기반 Windows 콘솔 프로그램으로 구현한다.
- 서버와 클라이언트를 별도 프로그램으로 분리한다.
- 서버와 클라이언트는 소켓 통신을 사용한다.
- 학생정보는 서버에서 연결 리스트로 관리한다.
- `users.txt`와 `SMU_Students.txt`는 탭 문자 구분 파일로 처리한다.
- 매일 01:00 학생정보 저장에는 `time.h`를 사용한다.

## 3. 파일 분리 원칙

| 파일 | 원칙 |
|------|------|
| `client_main.c` | 클라이언트 실행 진입점 |
| `client_ui.c` | 콘솔 메뉴와 서버 결과 출력 |
| `server_main.c` | 서버 실행 진입점 |
| `request_handler.c` | 요청 메시지 해석과 명령 분기 |
| `protocol.h` | 메시지 포맷, 명령 코드, 응답 상태 |
| `student.c/h` | 학생 구조체와 연결 리스트 |
| `user.c/h` | 사용자 구조체, 로그인, 사용자 추가/삭제 |
| `file_io.c/h` | 파일 입출력 |
| `scheduler.c/h` | `time.h` 기반 01:00 저장 |

## 4. 탭 구분 파일 처리

- `users.txt`는 사용자ID, 사용자암호, 사용자 소속부서, 사용자 이름 순서로 저장한다.
- `SMU_Students.txt`는 이름, 학번, 소속학과, 학년, GPA, 이메일 연락처, 모바일 폰 연락처 순서로 저장한다.
- 파일 파싱과 저장은 동일한 필드 순서를 사용한다.

## 5. 메시지 처리

- 메시지는 `TYPE<TAB>COMMAND<TAB>STATUS<TAB>PAYLOAD` 형식을 따른다.
- 요청 메시지는 `TYPE=REQ`, `STATUS=-`를 사용한다.
- 응답 메시지는 `TYPE=RES`, `STATUS=OK` 또는 `FAIL`을 사용한다.
