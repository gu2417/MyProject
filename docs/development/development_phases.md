# 개발 단계

## Phase 0 — 기반 구조

**목표**: 프로젝트 구조와 공통 헤더를 준비한다.

| 작업 | 완료 기준 |
|------|------|
| `src/client`, `src/server`, `src/common` 구성 | 서버/클라이언트/공통 코드 위치 분리 |
| `protocol.h` 작성 | 요청/응답 코드와 공통 상수 정의 |
| `student.h`, `auth.h`, `file_io.h`, `validation.h` 작성 | 구조체와 함수 원형 분리 |

## Phase 1 — 데이터 구조와 파일 입출력

**목표**: 학생 연결 리스트와 텍스트 파일 로딩/저장을 구현한다.

| 작업 | 완료 기준 |
|------|------|
| 학생 연결 리스트 구현 | 조회, 추가, 수정, 삭제 함수 작성 |
| 학생 파일 로딩 | `SMU_Students.txt`에서 10명 샘플 로딩 |
| 사용자 파일 로딩 | `users.txt`에서 3명 샘플 로딩 |
| 파일 저장 | 변경 사항이 파일에 반영 |

## Phase 2 — 인증과 기능 처리

**목표**: 서버 요청 처리와 핵심 기능을 연결한다.

| 작업 | 완료 기준 |
|------|------|
| 로그인 | 사용자 ID와 비밀번호 검증 |
| 사용자 추가/삭제 | `users.txt` 반영 |
| 학생 조회/추가/수정/삭제 | 연결 리스트와 파일 반영 |
| 학과명 조회 | 학과 기준 결과 반환 |
| 단과대학 검색 | 단과대학과 소속 학과 기준 결과 반환 |

## Phase 3 — 클라이언트와 통합

**목표**: 콘솔 메뉴와 소켓 통신을 연결한다.

| 작업 | 완료 기준 |
|------|------|
| 클라이언트 메뉴 | 로그인 전/후 메뉴 제공 |
| 요청 생성 | 메뉴별 서버 요청 생성 |
| 서버 응답 출력 | 성공, 실패, 조회 결과 출력 |
| 통합 테스트 | 정상/오류 시나리오 확인 |

## Out-of-Scope

|------|------|
