# 개발 단계

## Phase 0 — 구조 준비

| 작업 | 완료 기준 |
|------|------|
| 서버/클라이언트 폴더 분리 | 서버와 클라이언트 소스 파일 분리 |
| 공통 헤더 작성 | 메시지 포맷, 구조체, 함수 원형 분리 |
| 파일 경로 상수 정의 | `users.txt`, `SMU_Students.txt` 경로 관리 |

## Phase 1 — 데이터 관리

| 작업 | 완료 기준 |
|------|------|
| 학생 연결 리스트 구현 | 검색, 추가, 삭제, 수정, 학과 조회 가능 |
| 학생 파일 입출력 | `SMU_Students.txt` 로딩/저장 |
| 사용자 파일 입출력 | `users.txt` 로딩/저장 |

## Phase 2 — 서버 기능

| 작업 | 완료 기준 |
|------|------|
| 로그인 처리 | `users.txt` 기준 검증 |
| 사용자 추가 | 로그인된 사용자만 신규 사용자 추가 |
| 사용자 삭제 | 삭제 결과 `users.txt` 반영 |
| 학생정보 관리 | 연결 리스트와 파일에 반영 |
| 01:00 저장 | `time.h` 기반 저장 시점 처리 |

## Phase 3 — 클라이언트 및 통합

| 작업 | 완료 기준 |
|------|------|
| 콘솔 메뉴 | 사용자 편의 메뉴 제공 |
| 소켓 송수신 | 메시지 포맷 기반 요청/응답 처리 |
| 결과 출력 | 서버 결과를 클라이언트 콘솔에 출력 |
| 통합 확인 | 로그인 후 학생정보 관리 기능 사용 |
