# 계정 기능

## 1. 로그인

| ID | 기능 | 설명 |
|----|------|------|
| FR-AUTH-001 | 로그인 입력 | 학생 관리자는 클라이언트에서 사용자ID와 사용자암호를 입력한다. |
| FR-AUTH-002 | 로그인 요청 | 클라이언트는 로그인 요청을 서버에 송신한다. |
| FR-AUTH-003 | 로그인 검증 | 서버는 `users.txt` 의 사용자ID와 사용자암호를 기준으로 검증한다. |
| FR-AUTH-004 | 권한 부여 | 인가된 사용자만 학생정보 관리 기능을 사용할 수 있다. |
| FR-AUTH-005 | 결과 반환 | 로그인 결과는 클라이언트에 반환된다. |

## 2. 신규 사용자 추가

| ID | 기능 | 설명 |
|----|------|------|
| FR-USER-ADD-001 | 권한 확인 | 로그인된 사용자만 신규 사용자를 추가할 수 있다. |
| FR-USER-ADD-002 | 정보 입력 | 사용자ID, 사용자암호, 사용자 소속부서, 사용자 이름을 포함한다. |
| FR-USER-ADD-003 | 파일 저장 | 사용자 정보는 탭 구분 형식으로 `users.txt` 에 저장한다. |
| FR-USER-ADD-004 | 중복 방지 | 사용자ID는 중복될 수 없다. |

## 3. 사용자 삭제

| ID | 기능 | 설명 |
|----|------|------|
| FR-USER-DEL-001 | 삭제 기능 | 사용자 삭제 기능을 제공한다. |
| FR-USER-DEL-002 | 퇴사자 고려 | 퇴사자 계정 삭제를 고려한다. |
| FR-USER-DEL-003 | 파일 반영 | 삭제 결과는 `users.txt` 에 반영한다. |
| FR-USER-DEL-004 | 식별 기준 | 삭제 대상 식별 기준은 요구사항 미정이다. |
