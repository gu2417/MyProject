# 파일 스키마

## 1. 개요

본 시스템은 데이터베이스 서버를 사용하지 않고 텍스트 파일로 데이터를 저장한다. 서버는 학생 정보와 사용자 정보를 파일에서 읽고, 변경 사항을 파일에 저장한다.

**기준 파일**

| 파일 | 내용 |
|------|------|
| `SMU_Students.txt` | 학생 정보 |
| `users.txt` | 사용자 계정 정보 |

구체적인 파일 레코드 구분자와 필드 순서는 `requirements.md`에서 요구사항 미정으로 정의되어 있다. 본 문서의 샘플은 구현과 테스트를 위한 권장 예시이며, 실제 구현 시 `protocol.h` 또는 파일 입출력 모듈에서 일관되게 확정한다.

## 2. 학생 정보 파일

권장 파일명:

```text
SMU_Students.txt
```

권장 레코드 예시:

```text
student_id<TAB>name<TAB>department<TAB>grade<TAB>gpa<TAB>email<TAB>phone
```

| 순서 | 필드 | 자료형 | 설명 | 검증 기준 |
|------|------|------|------|------|
| 1 | student_id | 문자열 | 학생 고유 학번 | 필수, 중복 불가 |
| 2 | name | 문자열 | 학생 이름 | 필수 |
| 3 | department | 문자열 | 소속 학과 | 필수 |
| 4 | grade | 정수 | 학년 | 일반적으로 1~4 |
| 5 | gpa | 실수 | 평균 학점 | 일반적으로 0.0~4.5 |
| 6 | email | 문자열 | 이메일 주소 | 기본 이메일 형식 확인 |
| 7 | phone | 문자열 | 휴대전화 번호 | 필수 |

## 3. 사용자 정보 파일

권장 파일명:

```text
users.txt
```

권장 레코드 예시:

```text
user_id<TAB>password<TAB>department<TAB>name
```

| 순서 | 필드 | 자료형 | 설명 | 검증 기준 |
|------|------|------|------|------|
| 1 | user_id | 문자열 | 로그인 ID | 필수, 중복 불가 |
| 2 | password | 문자열 | 로그인 비밀번호 | 필수 |
| 3 | department | 문자열 | 사용자 소속 부서 | 필수 |
| 4 | name | 문자열 | 사용자 이름 | 필수 |

## 4. 파일 처리 정책

| 상황 | 처리 기준 |
|------|------|
| 서버 시작 | 학생 파일을 읽어 연결 리스트 초기화 |
| 로그인 | 사용자 파일을 기준으로 인증 |
| 학생 추가/수정/삭제 | 연결 리스트와 학생 파일에 반영 |
| 사용자 추가/삭제 | 사용자 파일에 반영 |
| 저장 실패 | 사용자에게 실패 사유 안내 |
| 파일 파싱 오류 | 프로그램이 비정상 종료되지 않도록 처리 |
| 파일 없음 | 요구사항 미정 |

## 5. 초기 데이터 기준

| 파일 | 초기 데이터 |
|------|------|
| `SMU_Students.txt` | 학생 10명 |
| `users.txt` | 사용자 3명 |

## 6. 시간 정보

시간 정보를 기록하거나 표시해야 하는 세부 범위는 요구사항 미정이다. 시간 값이 필요한 경우 `time.h`를 사용한다.
