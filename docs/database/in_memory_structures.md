# 인메모리 구조

## 1. 개요

서버는 학생 정보를 연결 리스트로 관리한다. 사용자 정보를 메모리에서 관리하는 구체적인 자료구조는 요구사항 미정이다.

## 2. 학생 연결 리스트

```c
typedef struct Student {
    char student_id[20];
    char name[50];
    char department[80];
    int grade;
    double gpa;
    char email[100];
    char phone[30];
    struct Student *next;
} Student;
```

## 3. 학생 리스트 연산

| 연산 | 기준 |
|------|------|
| 로딩 | `SMU_Students.txt`를 읽어 노드 생성 |
| 조회 | 연결 리스트 순회 |
| 추가 | 중복 학번 검증 후 새 노드 삽입 |
| 수정 | 대상 식별 기준은 요구사항 미정, 노드 값 갱신 |
| 삭제 | 삭제 기준은 요구사항 미정, 노드 제거 |
| 저장 | 연결 리스트 내용을 파일에 기록 |

## 4. 사용자 정보

```c
typedef struct User {
    char user_id[40];
    char password[80];
    char department[80];
    char name[50];
} User;
```

| 항목 | 기준 |
|------|------|
| 인증 | `users.txt` 기준 |
| 추가 | 중복 ID와 빈 필드 검증 |
| 삭제 | 삭제 기준은 요구사항 미정 |
| 저장 | `users.txt`에 반영 |

## 5. 단과대학 정보

단과대학과 소속 학과 정보를 이용한 학생 검색 기능은 제공해야 한다. 단과대학 및 소속 학과 정보의 구체적인 데이터 구조와 초기값은 요구사항 미정이다.

## 6. 메모리 관리

- 학생 노드 생성 시 메모리 할당 실패를 확인한다.
- 학생 삭제 시 제거한 노드의 메모리를 해제한다.
- 서버 종료 시 연결 리스트 전체 메모리를 해제한다.
